# ga3_test リポジトリ運用方針

## ディレクトリ＝プロジェクト
- このリポジトリでは、**トップディレクトリ直下の各ディレクトリを 1 つのプロジェクト**として扱います。
- **各プロジェクトの開発ルールは、そのプロジェクト直下の `AGENT.md` を参照してください。**
  作業対象のプロジェクトに入ったら、まずそのディレクトリの `AGENT.md` を読んでから進めること。

## プロジェクト一覧
- `default_development/` — 静的サイト雛形（Pug + SCSS + Gulp + webpack）。ルールは [`default_development/AGENT.md`](default_development/AGENT.md) を参照。
  - 旧トップディレクトリの `AGENT.md`（フロントエンド開発ルール）は、このプロジェクト直下の `AGENT.md` へ移しました。
- ※その他のディレクトリは案件ごとに追加。各ディレクトリに固有の `AGENT.md` を置いて運用する。

## 共通方針
- このトップ `AGENT.md` には、リポジトリ全体に関わる運用方針のみを記載します。
- 個別プロジェクトの技術スタック・コーディング規約・ビルドコマンド等は、各プロジェクトの `AGENT.md` に記述してください。
